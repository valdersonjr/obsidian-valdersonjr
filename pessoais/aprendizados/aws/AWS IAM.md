# AWS IAM

IAM (Identity and Access Management) controla **quem** pode fazer **o quê** em **quais recursos** na AWS.

## Conceitos

- **Principals**: usuários, roles, serviços.
- **Policies**: regras em JSON (allow/deny) para ações e recursos.
- **Roles**: identidade “assumível” (ex.: por EC2, Lambda, humanos via SSO).
